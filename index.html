MapReduce

Framework for distributed computing

Paper authors: Jeffrey Dean and Sanjay Ghemawat
Presentation by Martin Bjeldbak Madsen

spMI e15, Aalborg University

Motivation

How do we effectively run an inherently simple, parallelizable computation on large datasets?

Large datasets: thousands of terabytes Allows users to focus on their computation. Have hundreds or thousands of commodity computers Want to abstract details such as handling of data distribution, machine faults, computation distribution, and load balancing?
A programming model for processing and generating large data sets. Users specify a map function [..] and a reduct function. Many real world tasts are expressible in this model.
